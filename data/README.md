# Data

## Назначение каталога

Каталог содержит локальные данные и производные артефакты, используемые в рамках лабораторной работы.

## Структура

### `sources/`

Локальные parquet-источники в hive-style layout:

- `source_name/partition_key='partition_value'/part-*.parquet`

Этот каталог используется notebook и тестами как входной слой.

### `warehouse/`

Локальный path-based parquet warehouse для Spark-сценариев.

Ожидаемое содержимое:

- `dim_sources`
- `dim_attributes`
- `load_log`
- `tech_source_partitions`
- `client_monthly_attrs_scd1`
- `client_daily_attrs_scd2`

### `warehouse_polars/`

Аналогичное локальное хранилище для Polars-сценариев.

### `ml/`

Каталог для вспомогательных ML-данных и связанных артефактов, если они используются отдельными notebook или
экспериментами.

### `data.zip`

Архивная упаковка данных репозитория. Требует осторожности при обновлении, поскольку может содержать тяжёлые
артефакты.

## Правила работы с данными

- production-прогоны не должны удалять существующие тяжёлые артефакты без явного основания;
- debug-очистка допустима только для отдельно выделенного debug-root;
- уже загруженные неизменённые source partitions не должны перечитываться и перезаписываться по бизнес-логике;
- изменение однажды обработанной source partition должно считаться ошибкой, а не тихим overwrite.
